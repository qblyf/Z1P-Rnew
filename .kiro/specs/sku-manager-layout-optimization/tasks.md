# SKU 管理器布局优化 - 实现任务

## 概述

本任务计划解决 SKUManager 组件的布局问题，通过重构 flexbox 布局来分离可滚动表格区域和固定页脚按钮。

## 任务列表

### 任务 1: 重构 SKUManager 组件布局结构

**描述**: 将组件分离为三个不同的区域：固定顶部（SPU 信息）、可滚动中部（表单和表格）、固定底部（操作按钮）

**验收标准**:
- [ ] 顶部区域有 `flexShrink: 0` 属性
- [ ] 中部区域有 `flex: 1` 和 `minHeight: 0` 属性
- [ ] 中部区域有 `overflowY: auto` 属性
- [ ] 底部区域有 `flexShrink: 0` 属性
- [ ] 三个区域正确堆叠

**关联需求**: 需求 1, 2, 3, 4

---

### 任务 2: 更新表格组件

**描述**: 从表格中移除页脚，确保表格仅显示数据行

**验收标准**:
- [ ] 表格不包含页脚
- [ ] 表格头部保持 sticky 位置
- [ ] 表格行数据正确显示

**关联需求**: 需求 1, 2

---

### 任务 3: 实现固定底部操作按钮区域

**描述**: 在 SKUManager 中创建新的固定底部区域，放置"新增"和"确认修改"按钮

**验收标准**:
- [ ] 底部区域有上边框分隔
- [ ] 按钮有适当的间距和对齐
- [ ] 按钮样式与原有保持一致
- [ ] 底部区域不随表格滚动

**关联需求**: 需求 1, 3

---

### 任务 4: 测试不同 SKU 列表大小

**描述**: 验证按钮在各种 SKU 列表大小下都保持可见

**验收标准**:
- [ ] 0 个 SKU 时按钮可见
- [ ] 10 个 SKU 时按钮可见
- [ ] 100+ 个 SKU 时按钮可见
- [ ] 表格在内容超过视口时正确滚动

**关联需求**: 需求 1, 2, 3

---

### 任务 5: 测试按钮功能

**描述**: 验证表格滚动后按钮仍然可点击

**验收标准**:
- [ ] 表格滚动后"新增"按钮可点击
- [ ] 表格滚动后"确认修改"按钮可点击
- [ ] 表单控件在滚动时保持功能

**关联需求**: 需求 1, 3

---

### 任务 6: 验证响应式行为

**描述**: 测试组件在不同屏幕尺寸上的表现

**验收标准**:
- [ ] 手机屏幕上布局正确
- [ ] 平板屏幕上布局正确
- [ ] 桌面屏幕上布局正确
- [ ] 可滚动区域动态调整高度
- [ ] 间距和对齐保持一致

**关联需求**: 需求 4, 5

---

## 实现注意事项

1. **关键 CSS 属性**: `minHeight: 0` 是必需的，用于允许 flex 容器正确计算高度
2. **保留现有功能**: 仅修改布局，不改变组件逻辑
3. **保留样式**: 保留现有的自定义滚动条样式
4. **向后兼容**: 确保现有的 API 和 props 不变

## 完成标准

所有任务完成后，应满足以下条件：
- ✅ 操作按钮始终在视口内可见
- ✅ 表格可以正确滚动
- ✅ 布局在各种屏幕尺寸上都正确
- ✅ 所有按钮和表单控件都可正常使用
- ✅ 没有布局抖动或溢出问题
